# KUSOMEGANE アパレル商品管理ツール
## 自律開発エージェント向け 初回プロンプト

---

## このプロジェクトについて

KUSOMEGANEブランド（合同会社アリガトサン）のアパレル商品発売フローを自動化する社内管理ツールを開発します。

以下の3ファイルをすべて読んでから実装を開始してください：
- `01_PURPOSE.md` ── 目的・背景・解決したい課題
- `02_REQUIREMENTS.md` ── 機能要件・データモデル・外部連携仕様
- `03_TECHNICAL_SPEC.md` ── 技術スタック・実装コード・UI仕様

---

## Phase 0（まず完成させること）

以下をすべて実装してください。

### 1. Next.js + Tailwind + TypeScriptのプロジェクトセットアップ

```bash
npx create-next-app@latest . --typescript --tailwind --app --no-src-dir
npm install @anthropic-ai/sdk uuid
```

### 2. 実装する画面

**① ホーム画面（/）**
- 2カラムの商品カードグリッド
- 各カードに商品画像（上半分・背景はカラーに合わせた色）
- 画像上端オーバーレイ：商品番号・ステータスバッジ（未着手/対応中/完了）
- 画像下端：8分割進捗バー（完了分は #FFD600）
- カード下部：商品名・シリーズ名・カラー・次のアクション・サンプルあと◯日
- 上部サマリー：総商品数/対応中/完了
- フィルター：全て/対応中/完了
- 「+新規追加」ボタン

**② 新規商品登録ウィザード（/products/new）**
- 4ステップウィザード（A:画像 → B:基本情報 → C:キャプション → D:確認保存）
- 画像アップロードはドラッグ&ドロップ対応
- AIで画像を解析 → 商品種別・カラー・加工を自動提案
- カラー複数選択→商品番号の枝番自動採番（例: 59-1, 59-2）
- キャプション自動生成（Anthropic API）
- 保存時にimagePreviewをBase64で保存（ホームカードに表示するため）

**③ 商品詳細（/products/[id]）**
- ヒーロー画像（高さ200px）
- 8STEPタイムライン（各ステップの丸ボタンをクリックで完了トグル）
- STEP 5：サンプル到着予定日の入力欄
- STEP 7：素材チェックリスト（5項目）
- 生成済みキャプションのコピーボタン
- 商品情報テーブル

### 3. データ永続化
- LocalStorageを使用（lib/storage.tsの実装を03_TECHNICAL_SPEC.mdから使用）
- 初回表示時にサンプルデータ（2〜3件）を投入して動作確認できるようにする

### 4. Anthropic APIルート
- `/api/analyze-image` ── 画像解析
- `/api/generate-caption` ── キャプション生成
- APIキーは `.env.local` の `ANTHROPIC_API_KEY` から読む

### 5. ブランドカラー
- メインカラー: `#FFD600`（イエロー）
- 背景: 白・ライトグレー
- 完了色: `#22C55E`

## 重要な実装ポイント

- **画像表示バグに注意:** ウィザードで登録した商品のimagePreviewがホームカードに正しく表示されること。Base64のdata URLをそのままimgのsrcに渡す。
- **カラーバリエーション:** 複数カラー選択時はProductレコードをカラー数分作成し、productNumberに枝番（59-1, 59-2...）を付ける。ホームでは独立したカードとして表示。
- **Next.js App Router:** LocalStorage操作は必ず"use client"コンポーネント内で行うこと。
- **画像リサイズ:** アップロード時にcanvasで最大800pxにリサイズしてからBase64化する（LocalStorage容量対策）。

## 完成の定義（Phase 0）

以下がすべて動作すること：

1. ホーム画面に商品カードが表示される
2. 「+新規追加」から画像をアップロードして商品を登録できる
3. 登録した商品の画像がホームカードに表示される
4. Anthropic APIキーを設定すると画像解析・キャプション生成が動く
5. 商品詳細でSTEPをクリックして進捗を更新できる
6. STEP 7のチェックリストが機能する
7. リロードしてもデータが保持される（LocalStorage）
8. フィルター（全て/対応中/完了）が動く
