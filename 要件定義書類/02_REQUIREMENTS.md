# KUSOMEGANE アパレル商品管理ツール
## 要件定義書

**バージョン:** 1.0  
**作成日:** 2026-04-16

---

## 1. 現状フロー（AS-IS）

```
STEP 1  デザイン作成
STEP 2  商品合成作成（白背景・前後面を1枚に合成）
STEP 3  デザイン方向性確定 → Google Driveの商品番号フォルダに格納
STEP 4  スプレッドシートに手入力（8項目）→ メーカーに共有してサンプル依頼
STEP 5  サンプル到着確認（目安1週間・変動あり）
STEP 6  正式発注 & サンプルで物撮り
STEP 7  商品登録素材の準備
          ├── 商品画像
          ├── 加工箇所のアップ画像
          ├── 着画（AI生成）
          ├── サイズ詳細
          └── 商品概要キャプション
STEP 8  販売準備完了
```

---

## 2. 目標フロー（TO-BE）

```
STEP 1  デザイン作成（ツール外）
STEP 2  合成画像を作成（ツール外）
          ↓
【ツール】合成画像をアップロード + 最小限の情報入力
          ↓ 自動処理
          ├── AIが画像解析（商品種別・カラー・加工種別を自動判定・提案）
          ├── 商品番号を自動採番
          ├── 商品概要キャプションを自動生成
          ├── 準備物チェックリストを自動生成
          └── ホーム画面に商品カードが追加される

【ツール】Drive格納ボタン1クリック
          ↓ 自動処理
          ├── Drive内に商品番号フォルダを自動作成
          ├── 合成画像をアップロード
          └── スプレッドシートに行を自動追加（B〜H列 + 発注数量列）

STEP 4〜8  チェックリストをクリックするだけで進捗更新
```

## 3. 機能要件

### 3-1. ホーム画面（進捗ダッシュボード）

- **商品カードグリッド（2カラム）**
  - 上半分：商品合成画像（未登録時はプレースホルダー）
  - 画像上オーバーレイ：商品番号・ステータスバッジ
  - 画像下端：8分割進捗バー（完了分だけ #FFD600 で塗られる）
  - 下半分：商品名・シリーズ名・カラー・次のアクション・サンプル到着日カウントダウン
- **サマリー（3枠）:** 総商品数 / 対応中 / 完了
- **フィルター:** 全て / 対応中 / 完了
- **新規追加ボタン:** クリックで登録ウィザードへ遷移

### 3-2. 新規商品登録ウィザード（4ステップ）

#### STEP A：画像アップロード
- ドラッグ＆ドロップ または ファイル選択（PNG・JPG）
- アップロード後「AIで画像を解析する」ボタンを表示
- AI解析結果（商品種別・ボディカラー・デザイン要素・加工推定・雰囲気）を表示
- 解析スキップも可能

#### STEP B：基本情報入力フォーム

| フィールド | 型 | 自動/手動 | 備考 |
|---|---|---|---|
| 商品名 | テキスト | 手動・必須 | |
| シリーズ名 | テキスト | 手動 | |
| カラー | マルチセレクト | AI提案+手動 | 複数選択で自動枝番採番 |
| サイズ展開 | マルチセレクト | 手動 | S/M/L/XL/XXL/XXXL |
| 加工種別 | セレクト | AI提案+手動 | 刺繍/プリント/DTF/転写/刺繍+インク/刺繍+DTF |
| 加工指示 | テキストエリア | 手動 | スプシE列の内容 |
| ボディ型番 | テキスト | 手動 | CAB品番（例: 5001-01） |
| 素材 | テキスト | 手動 | デフォルト「綿100% 5.6oz」 |
| 受注生産フラグ | トグル | 手動 | デフォルトON |
| 送料無料フラグ | トグル | 手動 | デフォルトON |
| 備考 | テキストエリア | 手動 | スプシH列・メーカーへのサンプル指示等 |

#### STEP C：キャプション確認・編集
- AIが自動生成したキャプションを表示
- 編集可能なテキストエリア
- 「再生成」ボタンで別バリエーションを生成
- スキップ可能

#### STEP D：確認・保存
- 登録内容のサマリー表示
- アップロード画像のプレビュー
- 「商品を登録する」ボタンで保存
- Phase 1で自動化されること（Drive・スプシ連携）をメモとして表示

### 3-3. 商品詳細画面

- **ヒーロー画像エリア:** 登録済み合成画像を大きく表示、下部オーバーレイに商品名・番号・ステータス
- **8STEPタイムライン（ボックスリスト形式）**
  - 各行の左端の丸ボタンをクリックで「完了 ↔ 未着手」トグル
  - STEP 5（サンプル到着）：日付入力欄あり → ホームで「あと◯日」表示
  - STEP 7（素材準備）：以下のサブチェックリストが展開
    - 商品画像
    - 加工箇所アップ画像
    - 着画（AI生成）
    - サイズ詳細
    - 商品概要キャプション（自動生成済みの場合は最初からチェック済み）
- **生成キャプション表示：** コピーボタン付き
- **商品情報テーブル:** ボディ型番・素材・加工・受注生産・送料・備考

### 3-4. キャプション自動生成

#### 出力テンプレート（固定構造）

```
{受注生産フラグON}
※※※※※【この商品は受注生産商品です】※※※※※
{送料無料フラグON}
※※※※※【この商品に送料はかかりません】※※※※※
（空行）
{AI生成: 3〜5文のシリーズ説明+商品説明+購買促進文}

【商品情報】
カラー：{colors}
デザイン：{AI生成: デザイン1文説明}
素材：{material}
{加工種別が刺繍を含む場合}※デザインは刺繍加工です。

{受注生産フラグON}
【注意事項】
※受注生産商品になりますので、お届けまで約3週間程度いただいております。ご了承の上お買い求めください。
※生産状況によって早めにお届けになる場合もあります。
※ご注文商品の中で一番お時間のかかる商品に合わせて、発送スケジュールを組ませていただいております。
※欠陥品を除いて返品、交換は受け付けておりませんのでご理解の程お願いいたします。
```

#### AI生成部分の文体指示
- フレンドリー・ポップ・ファンと近い距離感
- 「〜です！」「〜してほしい！」調
- プレゼント・お揃い需要も自然に入れる
- 3〜5文・テンポよく読める長さ
- 参考文体：「ポチクソシリーズが、選べるようになりました！〜お揃いとかプレゼントとかでも楽しんでほしい！」

## 4. 外部連携要件

### 4-1. Google Spreadsheet

**対象シート:** `商品詳細（リスト生成）`  
**スプレッドシートID:** `1a6vkmUC_f7jPYirOwhUpbQcIXYvVjghbBc2fSr6X9wE`

#### カラム定義（確定版）

| 列 | 項目名 | 自動/手動 | 内容 |
|---|---|---|---|
| A | 商品（画像） | 手動 | セル画像（API非対応のため手動） |

| C | 色 | **自動** | ボディカラー |
| D | サイズ | **自動** | 例: `S/M/L/XL` |
| E | 加工 | **自動** | 種別＋指示内容 |
| F | ボディ型番 | **自動** | CAB品番 |
| G | デザインファイル | **自動** | DriveフォルダURL |
| H | 備考 | **自動** | メーカーへのサンプル指示 |
| I | S数量 | 手動 | 発注数量 |
| J | M数量 | 手動 | 発注数量 |
| K | L数量 | 手動 | 発注数量 |
| L | XL数量 | 手動 | 発注数量 |
| M | XXL数量 | 手動 | 発注数量 |
| N | 合計数量 | **自動（数式）** | `=SUM(I:M)` |

#### API操作
- **行追加:** カラーバリエーション1色ごとに1行追加
- **最大番号取得:** 起動時にB列を読んで次の商品番号を採番
- **発注数量更新:** I〜N列への書き込み

### 4-2. Google Drive

**ルートフォルダID:** `1jP6r0FSeSva_4z-XXiguKHzZ5uAYL1L1`

#### フォルダ構成ルール
```
/（ルートフォルダ）/
  ├── 15/       ← 商品番号でフォルダ管理
  ├── 16/
  └── ...
```

#### API操作
1. 商品番号フォルダが未存在 → 新規作成
2. 既存の場合 → そのまま使用
3. 合成画像を商品番号フォルダにアップロード
4. フォルダURLを取得 → スプシG列に書き込み

## 6. データモデル

### Product

```typescript
interface Product {
  id: string                        // UUID
  productNumber: string             // "59" or "59-1"
  baseProductNumber: number         // 59
  colorVariantIndex?: number        // 1, 2, 3...
  name: string                      // 商品名
  series: string                    // シリーズ名
  productType: string               // Tシャツ/パーカー/etc.
  colors: string[]                  // ["ブラック"]
  sizes: string[]                   // ["S","M","L","XL"]
  processingType: string            // 刺繍/DTF/etc.
  processingInstruction: string     // スプシE列
  bodyModelNumber: string           // スプシF列
  material: string
  isMadeToOrder: boolean
  freeShipping: boolean
  notes: string                     // スプシH列
  orderQuantities: {
    S?: number; M?: number; L?: number; XL?: number; XXL?: number
  }
  driveFolderUrl: string
  sheetRowNumbers: Record<string, number> // {"59-1": 5, "59-2": 6}
  captionText: string
  imagePreview: string | null       // Base64 or URL
  currentStep: number               // 1-8
  steps: FlowStep[]
  assets: ProductAssets
  sampleArrivalDate?: string        // ISO date string
  createdAt: string
  updatedAt: string
}

interface FlowStep {
  stepNumber: number                // 1-8
  status: "pending" | "in_progress" | "done"
  completedAt?: string
  notes: string
}

interface ProductAssets {
  compositeImage: "pending" | "uploaded" | "done"
  processingImage: "pending" | "uploaded" | "done"
  aiWearingImage: "pending" | "generated" | "done"
  sizeDetailDone: boolean
  captionDone: boolean
}
```

## 7. AI処理仕様

### 画像解析プロンプト（Anthropic Vision API）

```
この商品合成画像（白背景）を分析して以下をJSONのみで返答:
{
  "productType": "Tシャツ/パーカー/スウェット/トートバッグ/ロンT/キャップ/その他",
  "bodyColor": "色名（日本語）",
  "designElements": "デザインに含まれる要素の説明",
  "processingHint": "刺繍/プリント/DTF/転写/その他",
  "overallVibe": "全体の雰囲気（20字以内）"
}
```

### キャプション生成プロンプト

**システム:**
```
あなたはKUSOMEGANEブランドのEC担当者です。
フレンドリー・ポップ・ファンと近い文体で書いてください。
「〜です！」「〜してほしい！」調。JSONのみ返答。
```

**ユーザー:**
```
商品名: {name}
シリーズ: {series}
カラー: {colors}
加工種別: {processingType}
受注生産: {isMadeToOrder}
画像解析: {imageAnalysis}

出力:
{
  "description": "3〜5文のシリーズ説明+商品説明+購買促進文",
  "designDesc": "【商品情報】デザイン欄に入れる1文"
}
```

## 8. 非機能要件

- **対応デバイス:** PC・スマートフォン（レスポンシブ）
- **認証:** k2のみアクセス（Supabase Auth）
- **オフライン:** 非対応（常時ネット接続前提）
- **データ保存:** Phase 0はLocalStorage、Phase 1以降はSupabase
- **画像保存:** Phase 0はBase64/LocalStorage（5MB制限注意）、Phase 1以降はSupabase Storage or Drive
