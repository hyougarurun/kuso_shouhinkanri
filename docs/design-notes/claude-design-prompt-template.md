# Claude Design 投入テンプレート（kusomegane-apparel PC UI リニューアル用）

k2 の意図を Claude Design に**確実に**伝えるためのテンプレート集。

## 使い方

1. 下記「段階1」プロンプトをそのまま Claude Design のチャットにコピペ → 生成
2. 結果を確認 → 気に入らない点を 3 つ以内に絞って「段階1修正テンプレ」で指示
3. 骨格確定後、「段階2」で装飾・コンポーネント詳細
4. 最後に「段階3」でインタラクション

**ルール**: 1 回で完成を求めない。各段階で OK を出してから次へ。

---

## 段階 1: 骨格プロンプト（ホーム画面）

```
# 役割
あなたはシニア UI デザイナー。PC 専用業務ツールを Tailwind CSS ベースで設計する。

# 作るもの
KUSOMEGANE アパレルブランド向け商品管理ツールのホーム画面。
日本のアパレル事業者が使う、在庫管理的な堅さのある業務ツール。
PC 専用（モバイル非対応、メディアクエリ不要）。

# ユーザー像
- 1 人でアパレルブランドを運営する合同会社代表（1 人運用）
- 1 日に 10〜20 商品を登録 + 加工費を見積もる
- MacBook Pro を使う、同時に複数タブを開く
- 1 画面で「今日やること」と「商品一覧」が即わかる状態が理想
- 情報密度は高めで OK。親切心でアコーディオンや折りたたみを作らない

# 画面仕様（=変更不可）
- レイアウト: 左サイドバー 220px 固定 + 右メインエリア
- メインエリアは max-width: 1440px、中央寄せ、左右 padding 32px
- トップバー 48px（ロゴ + パンくず + 「+ 新規商品登録」ボタン）
- ホームの中身: 左右 2 カード横並び（grid-cols-3、左 span-2 / 右 span-1）
  - 左カード: 商品管理（フィルタタブ + サマリー + 商品カード 2 列グリッド）
  - 右カード: 加工費推定ツール（学習データ件数 + クイック推定フォーム + 最近の推定 5 件）

# 商品カード内容（ダミーデータ 6 件）
- サムネイル 120x120
- 商品番号（太字、黄色ハイライト 例: "#59"）
- 商品名（例: "KUSOMEGANE 花柄 T"）
- ステータスバッジ（例: "要確認"・"進行中"・"完了" の 3 種）
- 進捗バー（8 段階中 5 段階完了の表現）
- サンプル到着カウントダウン（例: "発送後 3 日目"）

# カラー
- 背景: #fafafa（zinc-50）
- カード・サイドバー背景: #ffffff
- 境界線: #e4e4e7（zinc-200）
- メインテキスト: #18181b（zinc-900）
- サブテキスト: #71717a（zinc-500）
- アクセント（CTA 押せるもの / アクティブナビ / 商品番号ハイライト）: #FBBF24（amber-400）
- 以上の 6 色のみ使用。他の色は禁止

# タイポ
- font-family: system-ui, -apple-system, "Hiragino Sans", "Noto Sans JP", sans-serif
- h1: 24px / bold
- h2: 18px / bold
- body: 14px / normal
- small: 12px / normal / text-muted
- line-height: 1.5

# コンポーネント詳細
- カード: rounded-lg / border 1px solid #e4e4e7 / bg-white / padding 24px / shadow-sm
- ボタン（CTA）: bg-[#FBBF24] text-black font-bold px-4 py-2 rounded-md hover:bg-[#F59E0B]
- ボタン（2次）: border text-zinc-700 bg-white hover:bg-zinc-50
- 入力: border rounded-md p-2 text-sm focus:ring-2 focus:ring-amber-400
- ステータスバッジ: 角丸 px-2 py-0.5 text-[11px] font-bold
  - 要確認: bg-red-100 text-red-700
  - 進行中: bg-blue-100 text-blue-700
  - 完了: bg-green-100 text-green-700

# 絶対に避けたいこと（=やらないで）
- グラデーション背景
- ガラスモーフィズム / ネオモーフィズム
- 絵文字による装飾（🎨✨等）
- モバイル対応（sm: md: lg: prefix の使用禁止）
- 過剰なアニメーション（hover transition 150ms 以下、scale/rotate 禁止）
- ダークモード対応
- 情報を隠すアコーディオン・タブ内タブ・ドロワー
- 「Dashboard」「Analytics」のような英語見出しの多用（日本語優先）
- AI らしい親切な余白過多

# 今回のアウトプット
ホーム画面 1 枚だけ、HTML + Tailwind CSS で。React コンポーネント形式も可。
他の画面（登録ウィザード等）は後で別プロンプトで依頼するので、今回は含めないで。
```

## 段階 1 修正テンプレ

結果が気に入らないときに使う。**3 点以内**に絞ること。

```
いま提示された案の以下 3 点だけ修正して、他は変えないで:

1. （具体的に何を、どう変えるか）
   例: 右カードの「加工費推定ツール」の見出しフォントが大きすぎる。h2 サイズ 18px に統一
2. （具体的に何を、どう変えるか）
   例: 商品カードのサムネイルが小さい。120px → 160px に
3. （具体的に何を、どう変えるか）
   例: トップバーのパンくずに絵文字ホームアイコンが入っているが、絵文字は禁止。削除

カラー・全体レイアウト・フォント階層は今のままキープ。
```

## 段階 2: 登録ウィザード Step 1（加工費推測画面）プロンプト

```
前回提示したホーム画面と同じデザイントークン（カラー・フォント・コンポーネント）で、
新規商品登録ウィザードの Step 1 画面を作って。

# 画面
- URL: /products/new
- 上部に進行バー（「1 加工費推測」現在 / 「2 登録情報」未 / 「3 登録完了」未）
- メインエリアに 1 つの大きなカード（p-8）
- カード内は 2 カラム（grid-cols-2、gap-8）

# 左カラム（画像）
- ドラッグ&ドロップ画像アップロード（破線 border-dashed border-zinc-300、hover でボーダー amber）
- アップロード後はプレビュー画像（max-height: 400px）
- 画像下に「画像を解析する」ボタン（2 次ボタンスタイル）
- 解析結果のリスト表示（検出した加工箇所・方法のテキスト箇条書き）

# 右カラム（入力）
- ボディ型番: input "5001-01"
- 色: select "ホワイト"
- 加工箇所（可変リスト）:
  - 1 行 = location select + method select + 削除ボタン
  - 「+ 加工箇所を追加」ボタン
- 最下部に「加工費を推定する」CTA ボタン（bg-black text-white、全幅）

# 下部（2 カラム下、fullwidth）
- 推定結果セクション（初期は非表示、ボタン押下後に展開）
  - ボディ単価レンジ 大きく表示
  - 加工費内訳テーブル（location / method / 単価 / 信頼度 / 件数）
  - 目立つ枠で「商品単価合計: ¥1,574 〜 ¥1,870」（bg-amber-50 border-amber-400）

# フッタ
- 左: "← キャンセル"（リンクテキスト）
- 右: "次へ →"（推定完了まで disabled、完了後 amber CTA）

# 禁止事項
- 段階 1 と同じ（グラデ・モバイル・絵文字等）
```

## リファレンス画像（k2 が用意するもの）

Claude Design は画像添付に対応している。以下の画像を添付すると大幅に精度が上がる:

1. **参考にしたい既存 UI**（例: Notion / Linear / Base 管理画面 / Shopify Admin 等のスクショ 2〜3 枚）
   - 「こういう雰囲気で」と言葉で補足
2. **避けたい UI**（例: 派手な SaaS ランディングページのスクショ 1 枚）
   - 「こういう装飾の多いのは違う」
3. **手書きワイヤー / napkin スケッチ**
   - k2 が描いた場合、撮影して添付
   - 言葉で描写するより 10 倍伝わる

## 対話テンプレ全体フロー

```
k2 → 段階 1 プロンプト + リファレンス画像 2-3 枚
Claude Design → 生成
k2 → 気に入った? (Y/N)
  N → 段階 1 修正テンプレ（3 点以内）
  Y → 段階 2 プロンプト
... (繰り返し)
k2 → 完成した HTML/CSS をコピー、もしくはスクショ
k2 → Claude Code（私）に共有 → 実装反映
```

---

## ワンポイント tips

- **「refine は 3 点まで」ルール**: 一度に多数を直すと AI が混乱して全体崩す
- **「他は変えないで」と明示**: デフォルトで AI は全体見直すので、範囲を絞る
- **数値は Tailwind のスケールに合わせる**: p-6（24px）、rounded-lg（8px）等、Tailwind の値を直接指定すると実装への移植が楽
- **「Hex 色を直接書いて」と指示**: Tailwind クラス名任せだと指定色と微妙にズレる
