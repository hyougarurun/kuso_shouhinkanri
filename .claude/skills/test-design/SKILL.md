docs/test-spec.md にテストケースを追加・更新しろ。

## 手順

1. `docs/test-spec.md` を読み込む
2. 対象機能に関連する既存テストケースを検索する
3. 以下を判断する:
   - 新規追加が必要か
   - 既存ケースの更新が必要か
   - 影響を受ける他のケースがあるか

## 新規テストケース追加時

以下の全項目を埋めること。1つでも空欄があれば追加しない。

- **テストケースID**: `TC-{カテゴリ}-{3桁連番}` （既存IDと重複しないこと）
- **テスト名**: 「{主語}が{動詞}する」形式
- **優先度**: P0（必須）/ P1（重要）/ P2（できれば）
- **テストデータ要件**: テストデータ一覧のIDを参照。新規データが必要なら先にテストデータ一覧に追加する
- **前提条件**: テスト実行に必要な全ての状態。「〜済みであること」だけでなく作り方も記載
- **UI実機パス**: 実際のルーティングパスとdata-testid。存在しない場合は代替手段も記載
- **手順**: 1ステップ1アクション。曖昧表現禁止
- **確認ポイント**: 具体的な期待結果
- **NG例**: テスターが迷うケースを具体的に
- **最終検証日**: 今日の日付

## 既存テストケース更新時

- 変更箇所を明示する（変更前 → 変更後）
- 最終検証日を今日の日付に更新する
- サマリーテーブルの数値も更新する
- 変更履歴テーブルに記録する

## 完了報告

追加・更新したテストケースを一覧で報告しろ:
```
テスト設計完了:
  追加: TC-XXX-001, TC-XXX-002
  更新: TC-XXX-003（手順変更）
  テストデータ追加: TD-005
```
