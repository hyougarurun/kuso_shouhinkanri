# Git ワークフロールール

## コミットルール

- コミットメッセージは日本語で記述
- フォーマット: `{prefix}: {変更内容の要約}`
- prefix一覧:
  - `テスト設計`: テスト仕様書の追加・更新
  - `feat`: 新機能の実装
  - `fix`: バグ修正
  - `refactor`: リファクタリング（機能変更なし）
  - `docs`: ドキュメントのみの変更
  - `chore`: ビルド・設定ファイル等の変更
  - `Phase X`: TDDワークフローのPhase完了
- 1コミット1変更。複数の変更をまとめない
- テスト設計と実装は別commitにする（TDDの証跡として）

## コミット前チェック

1. `git status` で変更内容を確認する
2. .envファイルが含まれていないことを確認する
3. テストが通っていることを確認する（テスト設計commitは除く）
4. `docs/progress.md` が最新であることを確認する

## ブランチ戦略

- main: 安定版
- feature/{機能名}: 機能実装ブランチ
- fix/{バグ名}: バグ修正ブランチ

## プッシュルール

- `git push --force` は禁止
- プッシュ前にテストが通っていることを確認する
- mainブランチへの直接pushはチーム開発時は禁止（PRを使う）
- 単独作業時はmain直接pushを許可（CLAUDE.local.mdで設定）

## .gitignore 必須項目

以下は必ずgit管理外にする:
- `.env` / `.env.*`
- `CLAUDE.local.md`
- `node_modules/`
- `.next/` / `dist/` / `build/`
